## Executive summary

The complaint — "drawings generated by the MCP/skills still look crude" — is **real but
narrower and more fixable than it appears**. The repository is **not** a dumb CRUD pass-through
that hand-draws everything from rectangles. There is a genuine server-side pipeline:

- `generateDiagramScene` → `layoutGraph` produces grid-snapped, layered, preset-styled cards
  with centered titles and bound, routed arrows (`backend/src/mcp/generate/diagram.ts:121`,
  `backend/src/mcp/layout/graphLayout.ts:143`).
- An auto-polish loop iterates geometric repairs until a score ≥ 95
  (`backend/src/mcp/quality/score.ts`, `autopolish.ts`).
- A library adapter can inject **real grouped vector glyphs** from official `.excalidrawlib`
  packs, normalize them, and reject insertions that worsen the score
  (`backend/src/mcp/libraries/libraryAdapter.ts:191`, `toolRegistry.ts:947`).

I confirmed this live: the architecture diagram *"Escola de Gente — Arquitetura do Sistema"*
already ships real **PostgreSQL / Redis / NestJS** icons, labeled bound arrows
(`HTTP/JSON`, `SQL`, `pool`, `cache`, `migrations`), a `dashed = async` convention, and a
structured **"Legenda & Notas"** panel. That is not crude.

So **why does it still read as a "technical scratch" rather than a portfolio piece?** Five
specific, evidence-backed reasons (full ranking in *Why current MCP output looks crude*):

1. **Real icons are never auto-injected.** The create path emits primitive cards only; icon
   insertion is an *opt-in, model-remembered* second tool call with **no concept→icon
   resolver** (`toolRegistry.ts:288` never calls `libraryAdapter.addItems`). Result:
   well-known tech (Postgres/Redis) gets a logo because the model bothered; generic nodes
   ("Visitante", "Equipe", "Web", "PgBouncer") stay iconless. **Inconsistent iconography is
   the single biggest crudeness signal.**
2. **Scoring is 100% geometry, blind to aesthetics.** `score = 100 − Σ severity penalties`
   (`score.ts:25`,`:171`). Nothing measures color harmony, contrast, palette discipline, icon
   richness, or typographic hierarchy. A flat, monochrome, all-rectangle diagram **scores ~100
   and passes**. There is zero upward pressure toward polish.
3. **No screenshot / raster review exists anywhere.** The renderer is never run during grading
   (repo-wide grep for `playwright|screenshot|getImageData` in `backend/src/mcp` = no hits).
   What the score grades is JSON coordinates, not the rendered image.
4. **Catalog is remote-only with no offline fallback.** Assets are fetched at runtime from the
   official GitHub repo (`validators.ts:19`). With no refresh the catalog is empty, curation is
   skipped, `cache_library` 404s, and **only primitives can be produced**.
5. **Style/typography tokens are prose, not enforced.** Presets, the 3-hue cap, WCAG AA
   contrast, 16px min font — all documented in `visual-system.md`/`geometry-rules.md`, none
   read by the quality module (and `minFontSize` is 14 in code vs 16 in docs).

The plumbing for professional output exists end-to-end. **The missing link is automation
(auto-pick + inject the right icon) plus aesthetic grading (a screenshot-based visual score
that punishes flat output and closes a repair loop).** This document specifies a *Visual
Quality Pipeline* and a 5-phase plan that reuses the existing layout/lint/repair/adapter and
the already-present Playwright e2e harness, rather than rebuilding.

## Current problem

The user perception: MCP/skills diagrams look like "rough technical drafts," not
"portfolio-ready art." The accurate framing after study:

- **What already works:** server-side layered layout, grid snapping, preset palettes, bound &
  routed arrows, centered titles, a 95-point geometric quality bar with hard blockers, optional
  legends, and real vector icon injection with a score-guard.
- **What is missing / crude:** iconography is *inconsistent* (only some nodes get real glyphs);
  colors look *per-node arbitrary* rather than disciplined by role/layer; the grid is *loose*
  (occasional edge crossings, notably ERD "spaghetti" connectors); the score cannot *see* any
  of this, so nothing pushes the model to fix it; and in any environment without a successful
  catalog refresh, output collapses to primitives entirely.

The problem is therefore **not "replace primitives with assets" but "make the existing
professional path the default, and add an aesthetic feedback loop that can perceive polish."**

## Excalidraw Libraries official experience

Captured live (`/tmp/excalidash-visual-study/01-official-libraries-home.png`,
`02-search-architecture.png`) and cross-checked against the upstream `libraries.json`:

- **Directory of 230 libraries**, MIT-licensed, one-click **"Add to Excalidraw"** + **Download**.
- **No formal taxonomy:** `libraries.json` is a flat array; each entry is
  `{name, description, authors, source, preview, created, updated, version, id}`. **Zero
  `tags`/`category` fields.** Discovery = keyword in name/description + SPA sort modes
  (`?sort=default|newest|most-downloaded`). Popularity (download counts, e.g. Software
  Architecture **201,634**) is the de-facto ranking signal.
- **Previews** are author-supplied **raster** snapshots (`.png`/`.jpg`) of the items laid out
  in a grid, stored next to the `.excalidrawlib` under `libraries/<author-slug>/`.
- **Items are real, grouped, reusable vector components:** a single library item is many
  primitives bound by a shared `groupId` with a stable name+id (v2), so it drops, moves, and
  **scales as one unit** while keeping the hand-drawn aesthetic.
- **Domain breadth out of the box** (the exact domains ExcaliDash targets): AWS, Azure, GCP,
  Kubernetes/Docker, C4, UML, networking, Kafka/MQ, dataflow.

**What the official ecosystem does that an MCP hand-building primitives does not:** real
recognizable iconography; consistent stroke/roughness/seed baked into every element; grouped
multi-element components; a human curation bar (≥3 related items, broadly useful, grouped,
English labels); stable identity & versioning (v1 array-of-arrays vs v2 `libraryItems`).

## Local library analysis

The two named local files were absent; analysis was performed on the **real upstream
equivalents** parsed from `excalidraw/excalidraw-libraries`.

### software-architecture.excalidrawlib
*(substitute: `youritjang/software-architecture` — the same set ExcaliDash's seed pack names
as "Software Architecture")*

- **7 items**, format **v1** (bare element arrays), **pure vector primitives, no embedded
  images.** Element mix: `line:11, ellipse:11, rectangle:8, freedraw:7, diamond:4`.
- Avg bbox **107×115 px**, **5.9 elements/item.** Intentionally **multi-color** (5 stroke
  colors, e.g. `#881fa3`, `#0a11d3`, `#c…`) because these are *diagram primitives*
  (microservice/database/cache/event-bus), meant to be tinted per node.

### drwnio.excalidrawlib
*(substitute: `drwnio/Software Logos` — `drwnio` is a real upstream author)*

- **18 items**, format **v1**, **no embedded base64** — logos are **reconstructed from many
  multi-point lines** (avg ~10.8 points/line, highest detail of the set). Element mix:
  `line:93, ellipse:12, rectangle:9, diamond:3, text:2, draw:1`.
- Avg bbox **443×410 px**, **6.7 elements/item**, `roughness: 1` (sketchy) — a *logo* set.

*(Cross-reference: `slobodan/AWS Serverless Icons` — 15 items, `line:96, ellipse:29,
rectangle:19, arrow:2`, avg **100×81 px**, **9.7 elements/item**, single stroke `#000000`,
`roughness: 0` — a clean **UI-icon** style, contrasting with drwnio's sketchy logo style.)*

**What makes these assets look polished (the rules to copy):**
1. **Grouping** — **39/40** parsed items carry `groupId`s → drop/move/scale as one object.
   *Single most important property.*
2. **Per-family style consistency** — one `strokeColor`, one `roughness`, one `fontFamily`
   per set (clean `roughness:0` for UI icons vs sketchy `roughness:1` for logos).
3. **Consistent bbox band** (~80–120 px for icons) so tiles compose at one scale.
4. **Native vector, never raster** — assets are redrawn as shapes, so they inherit the
   hand-drawn aesthetic and stay infinitely scalable/re-stylable.

## Codebase findings

### Library subsystem — `backend/src/libraries/*` + `backend/src/mcp/libraries/libraryAdapter.ts`
- **Remote-only catalog, no bundle.** Fetched at runtime from the allowlisted official host
  (`validators.ts:19` `ALLOWED_CATALOG_HOST = "raw.githubusercontent.com"`,
  `ALLOWED_CATALOG_PATH_PREFIX = "/excalidraw/excalidraw-libraries/main/"`). No `.excalidrawlib`
  or `libraries.json` ships in the repo or the `excalidash-v2-skills` npm package. Empty catalog
  ⇒ `packService.ts:178` `skippedReason: "Catalog is empty (no successful refresh yet)"`.
- **Seeds are pack-membership *names* only** (`corePack.ts:36` `{ name: "AWS Architecture
  Icons", aliases: ["aws","amazon web services"] }`), resolved against fetched rows; `Docker`
  is explicitly **absent** from the official catalog (`specializedPack.ts:82`).
- **Ranking is lexical, library-level, visually blind** (`ranking.ts:41` weights:
  exact-name 1000 … description 60; `BOOST_CURATED 200`, `BOOST_CATEGORY 80`). **No per-concept
  index** — the smallest ranked unit is a whole `.excalidrawlib`, never a single glyph; nothing
  reads aspect ratio/color/roughness of the asset.
- **Injection works but is caller-driven.** `libraryAdapter.addItems` (`:191`) clones an item's
  real `elements` with fresh ids + shared `groupId` + slot fitting + normalization, then
  appends to the scene (`:283`). MCP tools `add_library_items` / `add_library_items_normalized`
  (`toolRegistry.ts:502`); normalized mode **rejects** insertions that drop the score
  (`toolRegistry.ts:947`). But it needs explicit `libraryId` + `itemNames` and is gated on
  prior caching; `requireLibrary` defaults **false** (`lint.ts:77`,`:521`).

### MCP subsystem — `backend/src/mcp/*` (25 deterministic tools)
- **Route A (pure persistence):** `create_drawing` / `update_drawing` / `save_drawing` persist
  caller JSON after a **security-only** sanitize/redact (`server.ts:81`), with just an
  element-count cap (`toolRegistry.ts:93`) and permissive `z.any()` fields. **No layout, no
  styling, opt-in polish.** Quality = whatever the model typed.
- **Route B (server generation):** `create_diagram_from_prompt`, `create_from_template`,
  `create_from_repo_analysis`, `apply_architecture_skill`, `convert_diagram_type` →
  `generateDiagramScene` → `layoutGraph` (palette/stroke/font from preset, grid snap, bound
  arrows) → `polishAndMaybeSave` (auto-polish to ≥ 95). **This path is good and consistent.**
- `guide.ts:50` "The LLM is external; this MCP only executes 25 deterministic tools."
  `mcp-tool-cheatsheet.md:4` "there is no element-by-element API" (whole-scene granularity).

### Visual system & scoring — `backend/src/mcp/quality/{score,lint}.ts`, `_shared/references/*`
- **`score = clamp(0,100, 100 − Σ penalty)`**, `PENALTY = {error:9, warning:4, info:1}`
  (`score.ts:25`,`:171`). **Hard blockers** cap at 84 (`score.ts:66`; codes incl.
  `ARROW_TEXT_INTERSECTION`, `FRAME_TITLE_OVERLAP`, `TEXT_OVERFLOW`, `NO_LIBRARY_USAGE`).
  `passed = score ≥ 95 && 0 blockers` (`:209`). ~17 geometry detectors.
- **`DIMENSION_WEIGHT` is cosmetic** — computed for the breakdown, **never folded into the
  number** (`score.ts:41`,`:184`).
- **Aesthetics are unmeasured:** no color/contrast/harmony/hierarchy detector. **WCAG AA is
  documented but unimplemented** (`visual-system.md:56` vs no `luminance|contrastRatio|wcag` in
  code). Presets exist (`presets.ts:51` palette) but the quality module never imports them.
- **No raster scoring;** `visualRegression.test.ts` asserts lint codes on a JSON fixture only.
- **Doc/code drift:** `minFontSize:14` in code vs 16px in `geometry-rules.md:47`; documented
  spacing minimums (card 48 / frame 64 / lane 32) are prose, only `HIGH_DENSITY` (>10 / 220px
  cell) + 4px container padding are enforced; `requireLibrary/expectRichArchitecture/
  requireLegend` default off.

### Skills — `skills/excalidash/*` (25 skills + `_shared`)
- Skills restrict to `mcp__excalidash__*` and pick **one** Route-B create tool with `{nodes,
  edges}` + preset/direction; the **base diagram = primitives from the server layout engine**,
  not model-typed coordinates (`diagram.ts:127`).
- **Library art is a separate, manual, optional step;** the create path **never** calls
  `addItems` (`toolRegistry.ts:288` → `polishAndMaybeSave`, no injection). `library-curator`
  curates a catalog that no generation tool injects.
- **`visual-lint-repair-loop` is geometry-only and model-self-graded:** "pure geometric cleanup
  loop" (`SKILL.md:24`); the model self-tracks pass scores / rollback in prose
  (`examples.md:33` `score_drawing → 84 # > 78, keep pass`); `smoke.cjs` just prints the
  sequence (`process.exit(0)`).
- **No real per-type templates for most types** — ~13 generic node/edge lists all through the
  same `layoutGraph`; no microservices/sequence/dataflow scaffold.

### Frontend & e2e — `frontend/src/*`, `e2e/*`
- Renders with the real Excalidraw React canvas (`Editor.tsx`); the canvas selector is
  `canvas.excalidraw__canvas.interactive` (`drawing-crud.spec.ts:161`).
- **Exports SVG locally** via `exportToSvg` for thumbnails (`DrawingCard.tsx:143`) and the saved
  `preview` (`Editor.tsx:1105`); **no PNG/raster export in app code** — a PNG must come from a
  browser screenshot.
- **Playwright harness already exists and can run unauthenticated:** `global-setup.ts:61`
  onboards with `{enableAuth:false}`; `playwright.config.ts:70` boots backend+frontend;
  `docker-compose.e2e.yml` is self-contained; `Dockerfile.playwright` uses the Playwright image
  with browsers preinstalled; host browsers already downloaded.
- **Gaps for a visual loop:** `screenshot: "only-on-failure"`, no `toHaveScreenshot`/golden
  images, async render uses fixed `waitForTimeout` (flake risk), README port stale (5173 vs
  6767), local non-docker run needs a Postgres on `localhost:5432` + `npx playwright install`.

## Why current MCP output looks crude

Ranked root causes (each evidenced):

1. **Icons are never auto-injected** — create path is primitives-only; injection is an opt-in
   model-remembered call with no concept resolver (`toolRegistry.ts:288`, `diagram.ts:127`).
   *Largest single crudeness driver; produces "S3"/"Postgres" boxes instead of glyphs and the
   inconsistent iconography seen live.*
2. **Scoring is bbox-geometry only, blind to aesthetics** (`score.ts:25`,`:171`,`:66`). *A
   flat monochrome rectangle diagram scores ~100; no upward pressure to polish.*
3. **No screenshot/raster review in the quality path** (no `playwright|screenshot|getImageData`
   in `backend/src/mcp`). *Font fallback, in-slot icon fidelity, contrast, balance — all
   unverified.*
4. **Whole-library lexical ranking, no per-concept index, no visual-property awareness**
   (`ranking.ts:61`). *Even with the right pack cached, the model must guess the exact
   `itemName`, so glyph selection is unreliable.*
5. **Remote-only catalog, no bundled fallback** (`validators.ts:19`, `packService.ts:178`).
   *Offline/CI/first-boot ⇒ empty catalog ⇒ primitives only; the professional path is
   structurally unavailable.*
6. **Style/preset/typography tokens are prose, never scored; doc/code drift** (`presets.ts`
   not imported by quality; `visual-system.md:56` WCAG claim absent; `minFontSize` 14 vs 16).
7. **Route A bypasses the good engine** — `create_drawing/update_drawing/save_drawing` apply no
   layout/styling/polish (`server.ts:81`), so any hand-authored JSON gets the crudest output
   with no failing signal.

## Required product changes

- **Make professional output the default, not an expert path.** Recognized nodes should carry
  real glyphs automatically; the user should not need to know to ask for icons.
- **Ship a built-in icon substrate** so the product is never empty (offline/first-boot/CI).
- **Show quality to the user:** surface a combined geometry+visual score and a rendered review
  thumbnail in the dashboard/editor; expose a one-click "auto-polish (visual)".
- **Per-domain experience:** a preset/template/pack picker (AWS / Azure / GCP / microservices /
  C4 / dataflow), plus a legend toggle.
- **Honest degradation:** when no icon matches or library mode is off, fall back to clean
  primitive cards with no errors.

## Required technical changes

1. **Concept→item resolver** (`backend/src/libraries/conceptIndex.ts`): at cache time, index
   every `libraryItem` name into `concept → {libraryId, itemName, bbox, strokeColor,
   roughness}`. Promote `downloadService`'s flat `itemNames` (`downloadService.ts:106`) into a
   real index.
2. **Auto-injection in generation** (`generate/diagram.ts` + `libraryAdapter.addItems`): after
   `layoutGraph`, for each node with a resolved concept, inject the matching grouped glyph in
   normalized mode with the score-guard; unrecognized nodes keep the primitive card.
3. **Bundled offline fallback** + `packService` seed-from-bundle when refresh 404s (fix
   `packService.ts:178` short-circuit); parse both v1 and v2 (`libraryAdapter.ts:95-113`
   already does).
4. **Group generated widgets** — shared `groupId` for each card+text(+icon) in `graphLayout.ts`.
5. **Quality upgrades** (`quality/style.ts`, `lint.ts`, `score.ts`): `STYLE_DRIFT`,
   `LOW_CONTRAST` (real luminance/contrast), spacing-minimum detectors, `minFontSize`→16,
   `TYPO_HIERARCHY`; **fold `DIMENSION_WEIGHT` into the headline score**; enable
   `requireLibrary`/`requireLegend` for architecture presets.
6. **Screenshot pipeline** (`e2e/visual-capture.ts` + `quality/visualScore.ts` +
   `score_drawing_visual` tool): render canvas PNG, score aesthetics, blend with geometry.
7. **Server-side `run_repair_loop` tool** wrapping `autopolish.ts`+`repair.ts` with
   rollback-on-regression and new `INJECT_MISSING_ICON` / `NORMALIZE_STYLE` fixers.
8. **Per-domain bundles** (`templates/domainPacks.ts`) + a legend generator
   (`templates/legend.ts`).

## Proposed Visual Quality Pipeline

12 stages, each mapped to a concrete code location, test, and expected result.

| # | Stage | Does | Code location | Test | Expected |

## Library strategy

1. **Import + offline index (never empty):** build step fetches `libraries.json` + a curated
   subset of `.excalidrawlib` (AWS `childishgirl` 249-item v2, `youritjang` software-arch v1,
   Azure `7demonsrising` set, GCP, K8s, C4, data-flow) into a **versioned on-disk cache**;
   parse v1 + v2 (reuse `libraryAdapter.ts:95-113`). `catalogService.refresh` stays for online
   freshness but **seed from the bundle on 404** (fix `packService.ts:178`).
2. **Rank per domain *and* per concept:** keep `ranking.ts` for library-level selection; add
   `conceptIndex.ts` mapping `concept → {libraryId, itemName, bbox, strokeColor, roughness}`
   built at cache time, so "Postgres node" resolves to a specific glyph deterministically —
   **server-side, not by asking the model to guess `itemName`**.
3. **Inject as the default substrate:** generation auto-injects for recognized concepts in
   normalized mode (score-guarded); enforce `requireLibrary` for architecture presets so a
   rectangle-only architecture diagram actually fails.
4. **Style normalization on inject:** one stroke/roughness/font per family, consistent ~80–120px
   icon bbox, 8-digit alpha semantic fills — matching what makes upstream sets look polished.
5. **Licensing:** prefer runtime fetch from the allowlisted official host; bundle only a
   minimal, license-checked fallback subset (all upstream libs are MIT); record per-asset
   provenance (already tagged via `tagItemElements`).

## Playwright screenshot review strategy

Reuse the existing `e2e/` harness almost verbatim — every primitive needed already exists.

1. **Stack + auth:** `global-setup.ts` already disables auth (`{enableAuth:false}`), so a
   drawing can be created/opened with no login. Run local `webServer` mode (frontend 6767,
   backend 8000, local Postgres on 5432) or the self-contained `docker-compose.e2e.yml`.
2. **Open:** create via the API helper (`tests/helpers/api.ts createDrawing`) then
   `page.goto('/editor/'+id)` — the exact `drawing-crud.spec.ts` flow.
3. **Render-settle (kill flakiness):** Excalidraw has no completion signal; replace fixed
   `waitForTimeout` with a **stable-bytes poll** — screenshot twice ~300ms apart, proceed only
   when bytes are identical **and** non-white. Ensure `copy-excalidraw-assets.mjs` has run
   (fonts) or text renders in a fallback font and corrupts the score.
4. **Capture** the `canvas.excalidraw__canvas.interactive` region to PNG with a clean
   `{path, base64}` contract for the scorer.
5. **CI** via `Dockerfile.playwright` (browsers preinstalled) + `docker-compose.e2e.yml`; fix
   the stale README port (5173→6767).

## Scoring and repair strategy

Evolve in three layers without discarding the working geometry engine:

1. **Fix the existing score:** fold `DIMENSION_WEIGHT` into the headline number (today
   cosmetic, `score.ts:41`/`:184`); align `minFontSize` 14→16; enable
   `requireLibrary`/`requireLegend` for architecture presets so `RECTANGLE_ONLY` /
   `MISSING_LEGEND` actually bite.
2. **Add aesthetic geometry detectors (JSON-only, deterministic, cheap)** in `quality/style.ts`
   via `lint.ts`: `STYLE_DRIFT` (>3 hues / >1 accent / mixed roughness or font), `LOW_CONTRAST`
   (real luminance + contrast ratio — the WCAG AA check the docs claim but lack), and the
   documented spacing minimums. These flow through the existing severity machinery and
   immediately move the score.
3. **Add screenshot-based visual score** in `quality/visualScore.ts` consuming the Playwright
   PNG: (a) cheap deterministic pixel heuristics (global contrast, dominant-hue count,
   whitespace/ink balance, edge density as a busyness proxy) and/or (b) an LLM-judge for final
   polish. **Blend** into a combined score with the geometry **hard-blockers kept as a
   non-negotiable floor**. Make the loop a deterministic server tool (`run_repair_loop`) with
   best-kept + rollback-on-regression, not model self-grading.

## Risks and tradeoffs

| Risk | Tradeoff / mitigation |
|---|---|
| **Rendering cost** (boot browser + full stack; Excalidraw async render) | Run cheap geometry+style repairs first; re-screenshot **once per outer loop pass**, not per fix; make visual scoring opt-in for final polish; keep the geometry loop as the fast inner loop. |
| **Asset licensing** (bundling vendor icon sets redistributes IP) | All upstream libs are MIT; prefer runtime fetch from the allowlisted host; bundle only a minimal license-checked fallback; record per-asset provenance. |
| **Model token budget** (resolver + loop + image tokens) | Do concept resolution **server-side** (`conceptIndex`), not by asking the model to guess `itemName`; make the loop a single deterministic server tool so it costs one call, not a model-driven chain; LLM-judge optional. |
| **Screenshot flakiness** (no render-complete signal, fonts, layer differences) | Stable-bytes settle (two matching non-white frames) instead of fixed timeouts; assert non-blank before scoring; pin browser via `Dockerfile.playwright`. |
| **Aesthetic scoring is subjective / game-able** | Keep geometry hard-blockers as a non-negotiable floor; blend visual as a bounded fraction; calibrate against a small labeled fixture set; make the visual weight configurable/dial-down-able. |
